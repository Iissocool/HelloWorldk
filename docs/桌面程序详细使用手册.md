# NeonPilot 详细使用手册

## 1. 程序定位

NeonPilot 是一个个人使用定位的本地图像工作台。
它把抠图、批处理、批量命名、AI 生图和 Agent 控制台整合到同一个桌面程序里。

## 2. 主界面说明

### 仪表盘
- 查看硬件信息
- 查看推荐后端栈
- 查看模型目录
- 查看当前 Hermes 环境摘要

### 单图处理
- 适合先验证模型、后端和输出质量

### 固定批处理
- 整个目录统一用一个模型处理

### 智能批处理
- 按图片内容自动挑模型

### 批量命名
- `template`：按模板生成新文件名
- `replace`：查找替换
- `fresh`：忽略原文件名，直接全新覆盖命名

### AI 生图
- 自填 OpenAI 兼容接口地址
- API Key 加密保存
- 支持测试连接
- 支持批量生成并保存
- 支持程序内预览

### Agent
- 检测 WSL / Hermes 环境
- 导出 Hermes Skill
- 运行 `hermes --help`
- 运行 `hermes doctor`
- 使用程序自带 CLI 桥给 Hermes 发调用入口

### 任务历史
- 查看近期任务记录
- 查看 stdout / stderr / 报告路径

## 3. 批量命名说明

### template
示例：

```text
{index:03d}_{name}
```

变量：
- `{index}`：序号
- `{index:03d}`：补零后的序号
- `{name}`：原文件名
- `{parent}`：文件夹名
- `{ext}`：扩展名

### replace
- 查找文本：要被替换掉的内容
- 替换文本：新内容
- 如果替换文本留空，就等于删除这段文字

### fresh
- 基础名：比如 `product_`
- 起始序号：比如 `1`
- 序号位数：比如 `3`
- 最终会得到 `product_001`、`product_002` 这种名字

### 递归子目录
- 开启后，会继续处理所有下级文件夹里的目标文件

### 查找替换区分大小写
- 开启后，`old` 和 `Old` 会按不同文字处理

## 4. AI 生图说明

### 建议流程
1. 填写服务地址
2. 填写 API Key
3. 填写模型
4. 点击“保存配置”
5. 点击“测试连接”
6. 填写提示词和输出目录
7. 点击“开始生成”

### 安全说明
- API Key 使用当前 Windows 用户空间加密保存
- 程序不会把 Key 明文写入普通文本配置

## 5. Hermes 说明

### 当前可实现的部分
- 检测环境
- 导出 Skill
- 执行非交互命令
- 让 Hermes 调程序 CLI

### 当前不建议强做的部分
- 直接把 Hermes 的原生 TUI 完整嵌进 Tk 窗口

原因：
- Hermes 官方支持 Linux、macOS 和 WSL2
- 原生 Windows 不是主支持路径
- 完整终端体验更适合保留在 WSL 中运行

## 6. 动图背景

- 程序默认自带赛博朋克风格 GIF 背景
- 你也可以自己换成新的 GIF
- 点击顶部“更换动图背景”即可
- 点击“恢复默认背景”可回到内置背景

## 7. 窗口缩放

这版已经把左侧表单改成可滚动结构。
即使窗口被拖小，表单下半部分也依然可以通过滚动访问。

## 8. CLI 命令桥

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\run_neonpilot_cli.ps1 hardware
powershell -ExecutionPolicy Bypass -File .\scripts\run_neonpilot_cli.ps1 plan
```
