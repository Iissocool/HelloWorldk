# NeonPilot 详细使用手册

## 1. 程序定位

NeonPilot 是一个个人使用定位的本地图像工作台。
它把抠图、批处理、批量命名、AI 生图和 Docker Hermes 控制台整合到同一个桌面程序里。

## 2. 主界面说明

### 仪表盘
- 查看硬件信息
- 查看推荐后端栈
- 查看模型目录
- 查看当前 Docker Hermes 状态

### 单图处理
- 适合先验证模型、后端和输出质量

### 固定批处理
- 整个目录统一用一个模型处理

### 智能批处理
- 按图片内容自动挑模型

### 批量命名
- `template`：按模板生成新文件名
- `replace`：查找替换
- `fresh`：忽略原文件名，直接全新覆盖命名

### AI 生图
- 自填 OpenAI 兼容接口地址
- API Key 加密保存
- 支持测试连接
- 支持批量生成并保存
- 支持程序内预览

### Agent
- 启动 Docker
- 启动 Hermes
- 停止 Hermes
- 查看日志
- 导出 Skill
- 读取和保存模型配置
- 直接在程序里执行 Hermes 命令

### 任务历史
- 查看近期任务记录
- 查看 stdout / stderr / 报告路径

## 3. 批量命名说明

### template
示例：

```text
{index:03d}_{name}
```

变量：
- `{index}`：序号
- `{index:03d}`：补零后的序号
- `{name}`：原文件名
- `{parent}`：文件夹名
- `{ext}`：扩展名

### replace
- 查找文本：要被替换掉的内容
- 替换文本：新内容
- 替换文本留空时，表示删除这段文字
- 查找替换区分大小写：开启后，`old` 和 `Old` 会按不同文字处理

### fresh
- 基础名：比如 `product_`
- 起始序号：比如 `1`
- 序号位数：比如 `3`
- 最终会得到 `product_001`、`product_002` 这种名字

### 递归子目录
- 开启后，会继续处理所有下级文件夹里的目标文件

## 4. AI 生图说明

### 建议流程
1. 填写服务地址
2. 填写 API Key
3. 填写模型
4. 点击“保存配置”
5. 点击“测试连接”
6. 填写提示词和输出目录
7. 点击“开始生成”

### 安全说明
- API Key 使用当前 Windows 用户空间加密保存
- 程序不会把 Key 明文写入普通文本配置

## 5. Docker Hermes 说明

### 数据目录
Hermes 数据目录固定在：

```text
W:\gemini\data\neonpilot\hermes
```

这个目录会存放：
- `.env`
- `config.yaml`
- `skills`
- `sessions`
- `memories`
- `logs`

### Agent 页使用流程
1. 点击“刷新 Agent”确认状态
2. 如果 Docker 没启动，点击“启动 Docker”
3. 点击“启动 Hermes”准备后台服务
4. 需要看容器输出时点击“查看日志”
5. 需要把 Skill 给 Hermes 使用时点击“导出 Hermes Skill”
6. 需要修改默认模型时，直接在“模型设置”区域填写并保存
7. 在命令框里输入 `hermes --help`、`hermes doctor` 或 `hermes config show`
8. 点击“执行自定义命令”查看结果

### 不会再弹出的窗口
改成 Docker 路线后，程序运行 Hermes 命令时不会再弹出 Ubuntu 终端窗口。
命令会直接在程序内部执行并把输出写回 Agent 控制台。

## 6. 动图背景

- 程序默认自带赛博朋克风格 GIF 背景
- 你也可以自己换成新的 GIF
- 点击顶部“更换动图背景”即可
- 点击“恢复默认背景”可回到内置背景

## 7. 窗口缩放

这版已经把左侧表单改成可滚动结构。
即使窗口被拖小，表单下半部分也依然可以通过滚动访问。

## 8. CLI 命令桥

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\run_neonpilot_cli.ps1 health
powershell -ExecutionPolicy Bypass -File .\scripts\run_neonpilot_cli.ps1 hardware
powershell -ExecutionPolicy Bypass -File .\scripts\run_neonpilot_cli.ps1 plan
powershell -ExecutionPolicy Bypass -File .\scripts\run_neonpilot_cli.ps1 hermes-status
powershell -ExecutionPolicy Bypass -File .\scripts\run_neonpilot_cli.ps1 hermes-start
powershell -ExecutionPolicy Bypass -File .\scripts\run_neonpilot_cli.ps1 hermes-logs --tail 200
powershell -ExecutionPolicy Bypass -File .\scripts\run_neonpilot_cli.ps1 hermes-exec --text "hermes --help"
```

### 打包

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_windows.ps1
```

### 部署到 W:\gemini

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\deploy_w_gemini.ps1
```
