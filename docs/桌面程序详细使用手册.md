# NeonPilot 详细使用手册

## 1. 启动程序

仓库版：
```powershell
powershell -ExecutionPolicy Bypass -File .\run_desktop_app.ps1
```

说明：
- 启动脚本会自动创建 `.venv`
- 会自动安装最小桌面依赖
- 不会默认把全部抠图模型一次性下载完

## 2. 主界面说明

### 仪表盘
用途：
- 看硬件信息
- 看推荐后端
- 快速确认当前机器适合走哪条 GPU / CPU 路线

### 单图处理
用途：
- 先验证一张图
- 检查模型、后端、边缘质量

### 固定批处理
用途：
- 整个目录统一使用一个模型处理

### 智能批处理
用途：
- 让程序按图片特征自动选择模型

### 批量命名
模式：
- `template`：按模板命名
- `replace`：查找替换
- `fresh`：忽略原文件名，直接全新覆盖命名

界面中的简洁说明：
- `递归子目录`：继续处理下级文件夹里的目标文件
- `查找替换区分大小写`：开启后，`old` 和 `Old` 会被视为不同文本

### AI 生图
用途：
- 接入 OpenAI 兼容接口
- 保存 API Key
- 测试接口是否可用
- 生成图片并在右侧预览

说明：
- 服务地址使用固定舒适宽度，不会随着窗口拖动被拉得过长
- 右侧预览区会跟随窗口变大

### PS 套图
用途：
- 复用你已经做好的 Photoshop 模板
- 先打开指定 PSD
- 再把整个素材目录交给 Adobe Photoshop Droplet 自动处理

当前流程：
1. 选择 `模板 PSD`
2. 选择 `Droplet 程序`
3. 选择 `素材目录`
4. 确认 `Photoshop 程序`
5. 点击 `开始 Photoshop 套图`

常用按钮：
- `自动检测 Photoshop`
- `打开模板目录`
- `开始 Photoshop 套图`

说明：
- 当前桥接模块按你现有工作方式执行
- NeonPilot 负责“打开模板 + 把整个文件夹交给 Droplet”
- 最终保存位置仍由 Droplet 绑定的 Photoshop 动作决定
- 如果要把导出目录也交给程序统一控制，下一步需要换成 Photoshop 脚本桥接

### 资源中心
用途：
- 管理运行时组件
- 管理抠图模型

#### 运行时组件
- `core`：程序最小依赖
- `cpu`：CPU 抠图运行时
- `directml`：AMD / Intel 常用 GPU 路线
- `openvino`：Intel 专项路线
- `nvidia`：CUDA / TensorRT 路线

常用按钮：
- `刷新状态`
- `补齐最小运行依赖`
- `安装选中运行时`
- `卸载选中运行时`

#### 模型管理
常用按钮：
- `安装选中模型`
- `卸载选中模型`

说明：
- 只需要安装自己会用到的模型
- 不需要把全部模型一次性下载完
- 安装和卸载进度会显示在日志区和进度条里

### Agent
现在只保留三件事：
- 看状态
- 改配置
- 和 Hermes 聊天

#### 状态区
- `Docker`
- `Hermes`
- `对话能力`

#### 主要按钮
- `刷新状态`
- `一键准备 Agent`
- `查看日志`
- `打开数据目录`

#### 模型与 API 区
可设置：
- 提供方
- 默认模型
- 模型 Base URL
- API Key
- 提供方 Base URL

可用按钮：
- `读取配置`
- `保存配置`
- `测试 API`
- `测试聊天`
- `压缩配置`

#### 对话终端
使用方式：
1. 先完成 Agent 配置
2. 点击 `一键准备 Agent`
3. 在右侧输入消息
4. 点击 `发送消息`

说明：
- 不会再弹出 Ubuntu 终端窗口
- Hermes 对话直接留在程序内部
- 终端区域会随窗口放大

### 任务历史
用途：
- 回看最近任务
- 检查 `stdout / stderr / 报告路径`

## 3. 推荐首次使用顺序
1. 打开程序
2. 进入“资源中心”
3. 点击 `补齐最小运行依赖`
4. 需要 GPU 时，再按机器情况装 `directml / openvino / nvidia`
5. 安装你要用的模型
6. 回到“单图处理”先试一张图

## 4. 常见场景

### 只想先抠图
1. 在资源中心补齐最小依赖
2. 安装 `cpu` 或你需要的 GPU 运行时
3. 安装一个模型，例如 `u2netp`
4. 到单图处理页开始处理

### 想用 AMD / Intel GPU
1. 先装 `directml`
2. 再按需安装模型
3. 执行单图测试确认速度和效果

### 想用 Intel OpenVINO
1. 先装 `openvino`
2. 安装需要的模型
3. 在单图或批处理中手动切到 `openvino`

### 想用 NVIDIA
1. 先装 `nvidia`
2. 安装需要的模型
3. 在单图或批处理中选择 `cuda` 或 `tensorrt`

### 想和 Hermes 对话
1. 打开 Agent 页
2. 配好模型、Base URL 和 API Key
3. 点击 `测试 API`
4. 点击 `一键准备 Agent`
5. 在右侧终端直接发消息

### 想走 Photoshop 自动套图
1. 确认本机已安装 Photoshop
2. 准备好模板 PSD
3. 准备好可直接运行的 Droplet 程序
4. 打开 `PS 套图`
5. 先选模板和 Droplet
6. 再选素材目录并开始执行

## 5. 部署版位置
- 程序：`W:\gemini\apps\NeonPilot\NeonPilot.exe`
- 安装包：`W:\gemini\apps\NeonPilot\installer\NeonPilot-Setup.exe`
- 启动器：`W:\gemini\run_neonpilot.cmd`
- Hermes 数据目录：`W:\gemini\data\neonpilot\hermes`

## 6. 架构判断
当前不建议立刻把 Tkinter 全量替换成 Qt / Tauri / Electron。

原因：
- 当前性能瓶颈主要在推理运行时、模型安装和资源管理，不在 GUI 绘制
- 这轮已经把固定宽度表单、可伸缩预览区、可伸缩聊天区和资源中心补齐了
- 现在继续重写前端外壳，收益不如继续打磨运行时管理、更新机制和错误恢复

更适合的后续策略：
- 继续用现有桌面架构稳定迭代
- 优先补强安装器、更新器、日志诊断和资源下载体验
- 当界面复杂度继续明显上升时，再考虑迁移到更重的桌面框架
