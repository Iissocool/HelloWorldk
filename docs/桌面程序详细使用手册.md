# CutCanvas 使用手册

> 项目说明：这是一个以个人使用为定位、由纯 AI 辅助制作开发的项目，主要使用 GPT-5.4 进行设计、代码生成和迭代。当前项目仍在持续更新中。

## 1. 软件定位

这是一个 Windows-first 的本地抠图桌面程序。

目标是把 `rembg` 的多模型能力，结合 ONNX Runtime 的不同 Execution Provider，做成一套：

- 能识别硬件
- 能自动选路
- 能给 Intel / AMD / NVIDIA 分别走更合适后端
- 能做单图、批处理、智能批处理
- 能留存历史记录和完整日志

## 2. 支持的硬件路线

### Intel Windows

- 首选：`DirectML`
- 备选：`OpenVINO`
- 兜底：`CPU`

### AMD Windows

- 首选：`DirectML`
- 兜底：`CPU`

### NVIDIA Windows

- 首选：`TensorRT`
- 次选：`CUDA`
- 再次：`DirectML`
- 兜底：`CPU`

### Linux 专业路线

- AMD: `MIGraphX`
- Intel: `OpenVINO`
- NVIDIA: `TensorRT / CUDA`

说明：本仓库当前主要交付和测试重点是 Windows。

## 3. 首次安装

### 第一步：创建项目级 `.venv`

```powershell
py -3.12 -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip
.\.venv\Scripts\python.exe -m pip install -r requirements-app.txt
```

### 第二步：初始化运行时

普通 Windows AMD / Intel 用户：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\setup_windows_runtime.ps1
```

如果你还要准备 NVIDIA 环境：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\setup_windows_runtime.ps1 -IncludeNvidia
```

这个脚本会做这些事情：

1. 如有需要，克隆 `rembg` 源码
2. 应用本仓库附带的补丁
3. 创建桌面 App 所需 `.venv`
4. 创建 `venvs\rembg-dml`
5. 创建 `venvs\rembg-cpu`
6. 可选创建 `venvs\rembg-openvino`
7. 可选创建 `venvs\rembg-nvidia`

## 4. 启动方式

### 桌面程序

```powershell
powershell -ExecutionPolicy Bypass -File .\run_desktop_app.ps1
```

### Web 控制台

```powershell
powershell -ExecutionPolicy Bypass -File .\run_app.ps1
```

默认地址：

- `http://127.0.0.1:8787`

### 自检工具

```powershell
powershell -ExecutionPolicy Bypass -File .\run_self_test.ps1
```

## 5. 功能说明

### 仪表盘

显示：

- 系统
- CPU
- 内存
- GPU 列表
- 当前推荐后端栈
- 模型目录与个人使用提示

### 单图处理

你可以指定：

- 输入图路径
- 输出图路径
- 模型
- 后端

### 固定模型批处理

适合整批图片统一使用同一个模型。

### 智能批处理

会根据：

- 文件名关键词
- 图片色彩特征
- 边缘密度
- 皮肤占比
- 纹理复杂度

来判断更适合：

- 动漫模型
- 人像模型
- 细节模型
- 通用模型

### 任务历史

会记录：

- 任务时间
- 任务类型
- 后端
- 模型
- 输入/输出路径
- 是否成功
- 摘要
- stdout
- stderr
- 报告文件路径

## 6. 后端如何选择

### 推荐默认值

- Intel Windows: `auto`
- AMD Windows: `auto` 或 `amd`
- NVIDIA Windows: `auto`

### 后端含义

- `auto`: 由程序根据硬件自动决定
- `amd`: 显式 AMD Windows 路线，目前内部走 `DirectML`
- `directml`: Windows 通用 GPU 路线，尤其适合 AMD / Intel
- `openvino`: Intel 专项路线
- `cuda`: NVIDIA 通用 GPU 路线
- `tensorrt`: NVIDIA 高性能路线
- `cpu`: 兜底路线

## 7. AMD 用户特别说明

如果你的 AMD GPU 机器是 Windows：

1. 运行 `setup_windows_runtime.ps1`
2. 运行 `run_self_test.ps1`
3. 确认硬件画像里识别到了 AMD GPU
4. 启动桌面程序
5. 默认使用 `auto`
6. 如果你想强制走 AMD Windows 路线，可选择 `amd`

当前实现上：

- `amd` 会路由到 `DirectML`
- 这是 Windows 上对 AMD 最现实、部署成本最低的路线

## 9. 故障排查

### 报错 `No module named 'app'`

原因通常是：

- 没在项目根目录运行
- 启动脚本没有切到根目录

正确方式是运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\run_desktop_app.ps1
```

### AMD 机器没有吃到 GPU

先运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\run_self_test.ps1
```

检查：

1. 是否识别到了 AMD GPU
2. `available_backends` 里是否有 `directml`
3. 运行时是否已经通过 `setup_windows_runtime.ps1` 初始化

### OpenVINO 不可用

Intel 机器上如果 OpenVINO 环境没装完整，程序会回退到别的可用路线。

### NVIDIA 不可用

如果没有安装 NVIDIA 专用环境，`auto` 会走别的可用后端。

## 10. 个人使用提醒

当前项目定位是个人使用，但模型来源和相关条款仍建议自行了解清楚。

在更广泛分享或二次分发前，建议逐项检查：

- 模型权重许可
- 数据集许可
- 第三方运行时许可
- BRIA 等模型的官方条款范围
